# PerformanceMonitor 测试指南

## 如何测试 FPS 监控面板

### 1. 启动游戏
```bash
npm run dev
```

### 2. 显示 FPS 面板
按下快捷键：`Ctrl + Shift + P`

### 3. 验证项

#### 基本功能
- [ ] FPS 面板在左下角显示
- [ ] 显示三个指标：FPS、Avg、Min
- [ ] 数值实时更新
- [ ] 再次按 Ctrl+Shift+P 可以隐藏面板

#### 性能指标
- [ ] FPS 数值合理（通常 30-144）
- [ ] Avg FPS 低于或接近当前 FPS
- [ ] Min FPS 是最近 60 帧的最低值

#### 颜色指示
- [ ] FPS >= 50 时显示绿色
- [ ] FPS 30-49 时显示黄色
- [ ] FPS < 30 时显示红色

#### 性能影响
- [ ] 面板显示时游戏性能无明显下降
- [ ] 面板不影响控制和其他功能
- [ ] 面板不阻挡鼠标交互

### 4. 快捷键冲突测试
- [ ] Ctrl+Shift+P 不与其他快捷键冲突
- [ ] 游戏控制（方向键、空格等）正常工作

### 5. 构建验证
```bash
# 类型检查
npx tsc --noEmit

# 构建
npx vite build
```
- [ ] 类型检查通过
- [ ] 构建成功
- [ ] dist/ 目录生成

## 预期结果

所有测试项应该通过。FPS 面板应该：
- 轻量级（开销 < 0.5ms/帧）
- 准确（反映实际帧率）
- 非侵入式（不影响游戏体验）
- 可选（默认隐藏，用户可开启）
